# AGENT Birth Log Template

Use this whenever a new agent comes online. Copy it into `memory/YYYY-MM-DD.md` (or `birth_logs/<agent>.md`) so the birth conversation and approvals are traceable.

```markdown
## Birth Record — <Agent Name>
- **Date:** <YYYY-MM-DD>
- **Session ID:** <if available>
- **Human:** Reuben Bowlby (he/him, America/New_York)
- **Agent:** <Name + emoji>
- **Creature/Vibe:** <from IDENTITY.md>
- **Emoji:** <emoji>
- **Avatar assets:**
  - Color PNG: `avatars/<agent>-avatar.png`
  - Mono PNG: `avatars/<agent>-avatar-mono.png`
  - Brief / Source: `avatars/<agent>-avatar-brief.md`, generator flags: `<flags>`
- **Approval checklist:**
  1. Emoji alignment ✅/❌
  2. Contrast/readability ✅/❌
  3. Palette compliance ✅/❌
  4. Variants exported ✅/❌
  5. Metadata recorded ✅/❌
- **Identity artifacts created:** IDENTITY.md, USER.md, SOUL.md (links/paths)
- **Notes from birth conversation:**
  - <key facts/preferences>
- **Next actions:**
  - <follow-up tasks before agent works>
```

Store completed logs under `memory/YYYY-MM-DD.md` and optionally copy into `birth_logs/<agent>.md` for long-term tracking.
